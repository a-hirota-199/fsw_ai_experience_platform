# n8n — ワークフロー自動化（③〜⑧）

要件確定をトリガに、repo作成→Issue起票→スケルトン生成→PR→AIレビュー→ビルド→Slack通知 を連鎖させる。

## 原則（重要）

- **n8nは「サービス間の配線とトリガ」だけ**に限定する。ロジック（AI・外部呼び出しの中身）は backend のAPIに置き、n8nのノードは backend エンドポイントを順に叩くだけにする。
- ロジックが増えてもノードを足さず **backend API に寄せる**。
- イベント受信は**ポーリング既定**（ScheduleトリガでGitHub API等を定期取得）。Webhook＋トンネル(ngrok)はSlackボタンなどインタラクティブUIを入れる時のみ任意。

## 学習コスト対策

コミュニティ製 `czlonkowski/n8n-mcp`（MCP）を開発時に使うと、Claudeがn8nノード知識を持ち、ワークフローの検証・作成を補助できる。ただし公式ではなく、上の責務分離原則は維持する。

ワークフロー定義（エクスポートJSON）をこのディレクトリで管理する。
